# NLP-Translation

## Problem Description

This repository tackles a classic **Neural Machine Translation (NMT)** problem from English to Spanish using a **seq2seq** (Encoder-Decoder) architecture with **attention**. The main challenge was **overfitting**: the model memorized the training data without properly generalizing to new sentences. Throughout the iterations, various regularization, optimization, and data augmentation techniques were implemented to improve the model's generalization capability.

## Implemented Improvements

- **GPU / CUDA + CUDNN**: Drivers were installed to run training on GPU, drastically reducing computation time.

- **Early Stopping**: Keras callback that stops training when validation loss stops improving (patience of 3 epochs), preventing overfitting.

- **Dropout**: Dropout layers added to randomly deactivate neurons during training, functioning as a regularizer.

- **Pre-trained Embeddings (SpaCy)**: Pre-trained word vectors from SpaCy (`en_core_web_md` and `es_core_news_md`) were incorporated to initialize the embedding layer with rich semantic representations, allowing the model to better capture word relationships from the start of training.

- **ReduceLROnPlateau**: Adaptive learning rate reduction when the validation metric stagnates, enabling finer convergence.

- **L2 Regularization**: L2 penalty on the encoder and decoder LSTM layers to control weight magnitude.

- **Random Dataset Loading**: Random shuffling of sentences to prevent the model from learning sequential patterns from the data order.

- **Attention Mechanism**: Attention layer between the encoder and decoder that allows the model to focus on relevant parts of the source sentence when generating each word of the translation.

- **Second Dataset (Europarl)**: The current version of the repository has training implemented using `second_dataset.txt`, which was created from the **European Parliament Proceedings Parallel Corpus 1996-2011** (Europarl). This corpus is extracted from the proceedings of the European Parliament and includes versions in 21 European languages: Romance (French, Italian, Spanish, Portuguese, Romanian), Germanic (English, Dutch, German, Danish, Swedish), Slavic (Bulgarian, Czech, Polish, Slovak, Slovene), Finno-Ugric (Finnish, Hungarian, Estonian), Baltic (Latvian, Lithuanian), and Greek. This addition increased the dataset from ~144k sentences (`spa.txt`) to over **2 million lines** in total. The `creating_dataset.py` script combines the `europarl-v7.es-en.en` and `europarl-v7.es-en.es` files into `second_dataset.txt`.

> **Important note**: The results shown in the table do not include training with `second_dataset.txt`, as the computational cost increases dramatically with a dataset of this size and it was not possible to run it with the available hardware.
